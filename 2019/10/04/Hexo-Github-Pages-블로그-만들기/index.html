    이전에 블로그를 잘 운영하다가, 내 글 구려 병에 의해 블로그를 완전히 삭제하였다.

    블로그를 안하니, 공부한 내용을 제대로 정리하는 습관이 사라졌고…… 점점 바보가 되어 가는 것 같아 블로그를 다시 만들고자 하였다.

    뭔가 새로운 것은 없을까 하다 발견한 Hexo.

    이전에 사용하던 Jekyll은 루비 기반의 프레임워크였기 때문에 정이 가지 않았지만, Hexo는 Node.js 기반이기 때문에 재미있게 커스터마이징 할 수 있을 것 같아 Hexo로 블로그를 만들기로 결정!

    새로운 블로그의 첫 글은 Hexo를 이용한 블로그의 개설 방법으로 장식해보고자 한다.

    사전 준비

    그러므로 GitHub 아이디가 있어야 한다.

    또한 레포지토리를 만들어야 한다.

    레포지토리의 이름은 USERNAME.github.io로 하도록 한다.

    나의 경우, kimjangwook.github.io로 레포지토리를 생성하였다.

    요구사항

    매우 간단하니 아래의 링크를 통해 설치하길 바란다.

    커스텀 도메인을 가지고 있는 경우의 배포 방법

    2019-10-08 추가

    만약 커스텀 도메인이 적용되어있는 상태에서 위의 방법으로 디플로이를 할 경우 커스텀 도메인 설정이 해제될 것이다.

    이유는 커스텀 도메인 설정을 하게 되면 GitHub에 CNAME이라는 파일이 생성되는데, deploy를 하게 되면 CNAME 파일이 삭제되기 때문이다.

    해결방법은 간단하다.

    1. /source/ 폴더에 CNAME이라는 파일을 생성한다.

    2. CNAME 파일에 커스텀 도메인을 입력한다. 나의 경우에는 www.jangwook.net이라는 커스텀 도메인을 입력하였다. 파일 내용을 확인해보자.

    3. 이후 디플로이한다.

    위의 방법을 통하여 CNAME 파일까지 한번에 디플로이 하면 커스텀 도메인 설정이 해제되지 않는다.
